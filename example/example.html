# SlidePack themes
## A tale about defining some CSS classes

# SlidePack defines a very simple HTML structure

* The whole pack is an `<article>`
* Each slide is a `<section>` inside the `<article>`

And that's it.

--

# _per-slide_ CSS classes

To further customize your slides, you can apply any CSS
class to any slide using a special syntax in the markdown
source (_Ã  la_ [cleaver](https://github.com/jdan/cleaver))

    -- special awesome

    My special, awesome slide
    =========================

    This slide will have *special* and *awesome* CSS classes
    assigned

    --

    My normal, boring slide :(
    ==========================


--

# Default classes

SlidePack defines a series of CSS classes (or selectors)
with default styling:

* `section:first-child` for the first slide
* `section:last-child` for the last slide
* `section` for each section _"cover"_ (yeah, this is confusing...)
* `two-columns` displays lists in two columns
* `no-bullet` removes bullets from lists
  * use with caution when nesting lists
* `quote` for quoting someone
* `full-image` for a full-page image

## _quote_ slides need special care

Currently, the markdown library we use for processing
the slides does not support blockquote marks (>).

So, you have to use HTML :(

    <blockquote>
      "This. Is. Blockquote!"
      <cite>Some spartan</cite>
    </blockquote>

# Custom styles

SlidePack uses a default theme wich can be extended
using your own stylesheet. Just add it to the HTML,
like this:

    <link rel="stylesheet" type="text/css" href="…">

If you have problems getting your styles to work,
remember to check selector specificity!

--

# Remember

SlidePack is (almost) plain old HTML, so you can
use inline styles, javacript,… whatever! :D

--

# Total override

The default SlidePack theme is constrained like this:

    :not([class~=override]) {
    // theme styles here!
    }

So, you can disable it like this:

    <!-- add "override" as a CSS class to the html tag -->
    <html class="override">
    …

And even the whole SlidePack styles:

    <!-- remove the link to the stylesheet! :D -->
    <link rel="stylesheet" type="text/css" href="slide-pack.css">


--

# Power user

Define a hook called `_slide_pack_process_slides` and
process the slides as you see fit.  Remember to declare
the hook before you insert the `slide-pack` Javascript file.

    <script>
    var _slide_pack_process_slides = function ($slides) {
      // $slides is a Zepto.js wrapper containing all the slides
      $slides.wrap('<div class="woodoo"></div>');
    }
    </script>
    <script src="slide-pack.js"></script>
